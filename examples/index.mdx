## How examples are structured

Each example follows a consistent format:

1. **Problem statement** — what you need to solve
2. **Constraints** — limits or rules to follow
3. **Test cases** — sample inputs and expected outputs
4. **Solution** — fully working, commented code

This mirrors how problems appear in real interview settings and competitive programming.
