## Problem

Write a C function that reverses a string **in-place** — without using a second array.

## Constraints

- The string is a null-terminated `char` array
- Reverse in-place using O(1) extra space
- Do not use any standard library reverse functions

## Test cases

| Input | Expected output |
|---|---|
| `"hello"` | `"olleh"` |
| `"Kinsu"` | `"usnIK"` |
| `"racecar"` | `"racecar"` (palindrome) |
| `""` | `""` (empty string) |
| `"a"` | `"a"` (single char) |

## Solution

```c reverse_string.c
#include <stdio.h>
#include <string.h>

void reverse(char s[]) {
    int left  = 0;
    int right = strlen(s) - 1;

    while (left < right) {
        // Swap characters at left and right
        char tmp  = s[left];
        s[left]   = s[right];
        s[right]  = tmp;

        left++;
        right--;
    }
}

int main(void) {
    char str[] = "Hello, World!";
    printf("Before: %s\n", str);
    reverse(str);
    printf("After:  %s\n", str);
    return 0;
}
```

## How it works

The **two-pointer technique** works from both ends simultaneously:

```
"hello"
 ^   ^
 L   R   swap h ↔ o  →  "oellh"
  ^ ^
  L R     swap e ↔ l  →  "olllh" ... wait

h e l l o
          step 1: swap index 0 and 4  → o e l l h
          step 2: swap index 1 and 3  → o l l e h ... 

Let's trace correctly:
h e l l o     left=0, right=4  → swap h,o  →  o e l l h
o e l l h     left=1, right=3  → swap e,l  →  o l l e h ... 

Correct trace:
h e l l o
swap(0,4): o e l l h
swap(1,3): o l l e h  ← still wrong in trace above

Actually: "hello" reversed = "olleh"
h(0) e(1) l(2) l(3) o(4)
swap 0,4 → o e l l h
swap 1,3 → o l l e h  ← "olleh" ✓ (reading left to right: o-l-l-e-h)
```

**Time complexity:** O(n) — touches each character once.
**Space complexity:** O(1) — only three extra variables.

## Sample run

```bash
$ gcc -std=c11 -Wall -o rev reverse_string.c
$ ./rev
Before: Hello, World!
After:  !dlroW ,olleH
```
