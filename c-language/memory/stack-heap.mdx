## Two kinds of memory

Every C program uses two main regions of memory:

| | Stack | Heap |
|---|---|---|
| Managed by | Compiler / runtime automatically | You — via `malloc`/`free` |
| Size | Limited (typically 1–8 MB) | Large (limited by system RAM) |
| Allocation speed | Very fast (just moves a pointer) | Slower (traverses free list) |
| Lifetime | Tied to the enclosing scope/function | Until you call `free()` |
| Fragmentation | None | Can fragment over time |

## The Stack

Local variables, function parameters, and return addresses live on the stack. Memory is allocated when a function is called and **automatically reclaimed** when it returns.

```c
void foo(void) {
    int x = 10;      // on the stack
    int arr[100];    // also on the stack — 400 bytes
}
// x and arr are gone the moment foo() returns
```

The stack grows and shrinks like a... stack. Fast, automatic, and limited.

**Stack overflow** happens when the stack runs out of space — typically from infinite recursion or very large local arrays.

```c
void infinite(void) {
    int buffer[100000];  // 400 KB on the stack, every call
    infinite();          // will crash with a stack overflow
}
```

## The Heap

The heap is a large pool of memory for data that needs to outlive a function, or whose size is not known at compile time.

You request heap memory with `malloc` and return it with `free`.

```c
#include <stdlib.h>

int *arr = malloc(10 * sizeof(int));  // 40 bytes on the heap
if (arr == NULL) {
    // malloc failed — always check!
    return -1;
}

arr[0] = 42;  // use it like a normal array

free(arr);    // return memory to the heap
arr = NULL;   // good practice — prevents use-after-free
```

## Common mistakes

<AccordionGroup>
  <Accordion title="Returning a pointer to a stack variable">
    ```c
    int* broken(void) {
        int x = 42;
        return &x;   // DANGER: x is destroyed when function returns
    }
    ```
    The memory `&x` points to is reclaimed immediately. Use heap allocation instead.
  </Accordion>
  <Accordion title="Stack overflow from large local arrays">
    ```c
    void process(void) {
        int huge[1000000];  // ~4 MB — likely crashes
    }
    ```
    For large buffers, allocate on the heap with `malloc`.
  </Accordion>
  <Accordion title="Forgetting to free heap memory">
    ```c
    void leak(void) {
        int *p = malloc(100);
        // ... use p ...
        // forgot free(p) — memory leak!
    }
    ```
    Leaked memory is not reclaimed until the program exits. In long-running programs, leaks can exhaust available memory.
  </Accordion>
</AccordionGroup>
