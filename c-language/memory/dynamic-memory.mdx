## The memory management functions

All live in `<stdlib.h>`:

| Function | Purpose |
|---|---|
| `malloc(size)` | Allocate `size` bytes, uninitialized |
| `calloc(n, size)` | Allocate `n * size` bytes, zero-initialized |
| `realloc(ptr, size)` | Resize a previous allocation |
| `free(ptr)` | Release allocated memory |

## `malloc`

```c
#include <stdlib.h>

// Allocate space for 10 ints
int *arr = malloc(10 * sizeof(int));

if (arr == NULL) {
    fprintf(stderr, "Memory allocation failed\n");
    return -1;
}

for (int i = 0; i < 10; i++) {
    arr[i] = i * 2;
}

free(arr);
arr = NULL;
```

<Warning>
  `malloc` does not initialize memory — contents are undefined. Always check for `NULL` before using the returned pointer.
</Warning>

## `calloc`

Allocates and **zero-initializes** all bytes:

```c
int *arr = calloc(10, sizeof(int));
// equivalent to malloc(10 * sizeof(int)) + memset to 0
```

Prefer `calloc` when you need a zero-initialized buffer. It is also safer against integer overflow in the size calculation.

## `realloc`

Resize an existing allocation. Common pattern for dynamic arrays:

```c
int capacity = 4;
int size = 0;
int *arr = malloc(capacity * sizeof(int));

// When full, double the capacity
if (size == capacity) {
    capacity *= 2;
    int *tmp = realloc(arr, capacity * sizeof(int));
    if (tmp == NULL) {
        free(arr);  // realloc failed but arr is still valid
        return -1;
    }
    arr = tmp;
}
```

<Warning>
  Never assign `realloc`'s return value directly to the original pointer. If `realloc` fails, it returns `NULL` but does NOT free the old block. Using a temporary variable prevents a memory leak.
</Warning>

## `free`

Returns memory to the heap so it can be reused:

```c
free(ptr);
ptr = NULL;  // prevent accidental use-after-free
```

Rules for `free`:
- Only free memory returned by `malloc`, `calloc`, or `realloc`
- Never free the same pointer twice (double-free is undefined behavior)
- Never free a pointer to stack memory
- Set the pointer to `NULL` after freeing

## Detecting memory errors

Use **Valgrind** (Linux/macOS) or **AddressSanitizer** to catch leaks, use-after-free, and double-free at runtime:

```bash
# Compile with AddressSanitizer
gcc -std=c11 -fsanitize=address -g -o program main.c

# Run — it will report any memory errors
./program
```

```bash
# Valgrind memory leak check
valgrind --leak-check=full ./program
```

## A complete dynamic array example

```c
#include <stdio.h>
#include <stdlib.h>

typedef struct {
    int *data;
    int size;
    int capacity;
} DynArray;

DynArray* da_create(int initial_cap) {
    DynArray *da = malloc(sizeof(DynArray));
    da->data = malloc(initial_cap * sizeof(int));
    da->size = 0;
    da->capacity = initial_cap;
    return da;
}

void da_push(DynArray *da, int value) {
    if (da->size == da->capacity) {
        da->capacity *= 2;
        da->data = realloc(da->data, da->capacity * sizeof(int));
    }
    da->data[da->size++] = value;
}

void da_free(DynArray *da) {
    free(da->data);
    free(da);
}

int main(void) {
    DynArray *arr = da_create(4);
    for (int i = 0; i < 10; i++) da_push(arr, i * i);
    for (int i = 0; i < arr->size; i++) printf("%d ", arr->data[i]);
    da_free(arr);
    return 0;
}
```
