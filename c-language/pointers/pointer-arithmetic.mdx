## How pointer arithmetic works

When you add an integer to a pointer, C automatically scales the offset by the **size of the pointed-to type**:

```c
int arr[] = {10, 20, 30, 40, 50};
int *p = arr;  // points to arr[0]

p + 1  // points to arr[1] — advances by sizeof(int) bytes (4)
p + 2  // points to arr[2] — advances by 8 bytes
```

So `p + n` does not mean "n bytes forward" — it means "n elements forward."

## Incrementing and decrementing

```c
int arr[] = {10, 20, 30, 40};
int *p = arr;

printf("%d\n", *p);    // 10
p++;
printf("%d\n", *p);    // 20
p++;
printf("%d\n", *p);    // 30
p--;
printf("%d\n", *p);    // 20
```

## Traversing an array with a pointer

```c
int arr[] = {1, 2, 3, 4, 5};
int *end = arr + 5;  // one past the last element

for (int *p = arr; p < end; p++) {
    printf("%d ", *p);
}
// 1 2 3 4 5
```

This is exactly how the C standard library iterates over ranges internally.

## Pointer subtraction

Subtracting two pointers gives the number of **elements** between them:

```c
int arr[] = {10, 20, 30, 40, 50};
int *p1 = &arr[1];
int *p2 = &arr[4];

ptrdiff_t diff = p2 - p1;  // 3
printf("%td elements apart\n", diff);
```

The result type is `ptrdiff_t` from `<stddef.h>`.

## Pointer comparison

Pointers to elements of the same array can be compared with `<`, `>`, `==`:

```c
int *p = arr;
int *q = arr + 3;

if (p < q) printf("p comes before q\n");
```

## The `void *` pointer

`void *` is a generic pointer — it can point to any type without a cast. Used extensively in functions like `malloc` and `memcpy`:

```c
void *ptr = malloc(100);  // malloc returns void*
int *ip = ptr;            // implicitly cast to int* in C (no cast needed)
```

You cannot dereference a `void *` directly — you must cast it to a concrete type first.

## `const` with pointers

There are two different things you can make constant:

```c
const int *p = &x;    // pointer to const int — cannot modify *p
int *const p = &x;    // const pointer to int — cannot change p itself
const int *const p;   // both are const
```

This distinction matters a lot in function signatures — when a function should not modify the data it receives through a pointer, declare the parameter `const`:

```c
void print_array(const int *arr, int len) {
    for (int i = 0; i < len; i++) {
        printf("%d ", arr[i]);
        // arr[i] = 0;  // ERROR — cannot modify through const pointer
    }
}
```
