## The C compilation model

Unlike interpreted languages (Python, JavaScript), C is **compiled** — your source code is transformed into machine code before it ever runs.

```
Source (.c) → Preprocessor → Compiler → Assembler → Linker → Executable
```

This gives C its speed, but also means you interact with the build process directly.

## The four stages of compilation

<Steps>
  <Step title="Preprocessing">
    The preprocessor handles `#include`, `#define`, and other directives. It runs before the compiler sees your code.
  </Step>
  <Step title="Compilation">
    The compiler (`cc` or `gcc`) converts your C source into assembly language specific to your CPU architecture.
  </Step>
  <Step title="Assembly">
    The assembler converts assembly into object files (`.o`) containing raw machine instructions.
  </Step>
  <Step title="Linking">
    The linker combines object files and resolves references (like `printf`) from system libraries to produce the final executable.
  </Step>
</Steps>

## C standards

C has gone through several standardized revisions. You will encounter these names often:

| Standard | Year | Notes |
|---|---|---|
| K&R C | 1978 | Original version from Kernighan and Ritchie's book |
| C89 / ANSI C | 1989 | First official standard |
| C99 | 1999 | Added `bool`, `//` comments, variable-length arrays |
| C11 | 2011 | Added threads, atomic operations, better Unicode |
| C17 | 2018 | Bug fixes over C11, most compilers default to this |
| C23 | 2023 | Latest standard, adds `nullptr`, `bool` as keyword |

<Tip>
  For most learning purposes, **C99** or **C11** is a solid target. Use `gcc -std=c11` to compile with C11 features.
</Tip>

## What C does not have

Understanding what C lacks helps set expectations:

- No built-in garbage collection — you manage memory manually
- No built-in string type — strings are arrays of `char`
- No classes or objects — C is procedural, not object-oriented
- No namespaces — everything lives in one global space
- No exception handling — errors are handled via return codes

These are not weaknesses; they are deliberate design choices that give C its power and portability.
