## The `if` statement

The most fundamental control structure. A block executes only when its condition is true (non-zero).

```c
int score = 85;

if (score >= 90) {
    printf("Grade: A\n");
}
```

## `if` / `else`

```c
int x = 10;

if (x > 0) {
    printf("Positive\n");
} else {
    printf("Zero or negative\n");
}
```

## `if` / `else if` / `else`

Chain multiple conditions — only the first true branch executes:

```c
int score = 72;

if (score >= 90) {
    printf("A\n");
} else if (score >= 80) {
    printf("B\n");
} else if (score >= 70) {
    printf("C\n");
} else {
    printf("F\n");
}
```

## Omitting braces

Single-statement branches can omit `{}`, but this is discouraged:

```c
// Works, but error-prone
if (x > 0)
    printf("Positive\n");

// Always prefer explicit braces
if (x > 0) {
    printf("Positive\n");
}
```

<Warning>
  The "dangling else" problem is a classic C bug. Always use braces to make the pairing unambiguous.
</Warning>

## Nested conditionals

```c
int age = 20;
int has_id = 1;

if (age >= 18) {
    if (has_id) {
        printf("Access granted\n");
    } else {
        printf("ID required\n");
    }
} else {
    printf("Too young\n");
}
```

## The ternary operator

A concise one-line conditional for simple value assignments:

```c
int a = 5, b = 10;
int max = (a > b) ? a : b;
printf("Max: %d\n", max);  // 10
```

Avoid nesting ternary operators — it hurts readability fast.

## Truthy and falsy in C

C has no native boolean type before C99. Any integer value is used as a condition:

| Value | Treated as |
|---|---|
| `0` | false |
| Any non-zero | true |
| `NULL` pointer | false |
| Non-NULL pointer | true |

```c
int n = 5;
if (n) {           // same as if (n != 0)
    printf("n is non-zero\n");
}
```
